<h2 id='dod'> 🏆 Definition of Done (DoD) </h2>

O **DoD** é o nosso critério de "Pronto". Uma User Story só é considerada finalizada se atender a todos os requisitos abaixo:

| Critério | Descrição | Status | Evidência |
|:----------|:-----------|:---------:|:-----------|
| **Código Funcional** | A funcionalidade foi implementada e atende aos Critérios de Aceitação. | [ ] | [Link Demo](#) |
| **Versionamento** | Código commitado no Git seguindo o padrão: `tipo(#id): descrição`. | [ ] | [Git History](#) |
| **Branching** | Desenvolvido em branch própria: `feature/<nome-da-funcionalidade>`. | [ ] | [Branches](#) |
| **Code Review** | O código foi revisado por pelo menos um colega via Pull Request. | [ ] | [Pull Requests](#) |
| **Código Limpo** | Código segue o padrão do projeto, sem trechos comentados ou lixo. | [ ] | [Sonar/Lint](#) |
| **Testes Aprovados** | Testes unitários e manuais executados com sucesso. | [ ] | [Relatório](./docs/relatorio_testes.md) |
| **Documentação** | O arquivo de [User Stories](./docs/user_stories.md) e o README foram atualizados. | [ ] | [Docs](./docs/) |
| **Apresentação** | Incremento demonstrado ao PO/Professor na Review da Sprint. | [ ] | [Vídeo/Print](#) |
